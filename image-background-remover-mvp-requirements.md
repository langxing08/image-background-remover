# Image Background Remover 网站 MVP 需求文档

## 1. 项目背景

本项目面向有“图片去背景”需求的用户，提供一个在线工具网站。  
用户上传图片后，系统调用 Remove.bg API 自动去除背景，并返回透明背景结果图供用户预览和下载。

本期目标是以最低成本、最快速度上线一个可用版本，用于验证以下几点：

- 关键词 `image background remover` 是否有可获取流量
- 用户是否愿意直接使用在线抠图工具
- 页面体验、处理速度、效果是否满足基础预期
- 后续是否值得继续做 SEO、商业化和功能扩展

---

## 2. 项目目标

### 2.1 核心目标
上线一个可公开访问的最小可用版本（MVP），实现完整闭环：

- 上传图片
- 自动去背景
- 展示处理结果
- 下载透明 PNG

### 2.2 非目标
本期不做以下内容：

- 用户注册 / 登录
- 历史记录保存
- 数据库存储
- 对象存储
- 批量处理
- 支付与会员体系
- 后台管理系统
- 多语言
- API 对外开放

---

## 3. 目标用户

### 3.1 主要用户
- 有临时图片抠图需求的普通用户
- 电商图片处理用户
- 设计 / 内容创作者
- 需要快速生成透明背景图的运营人员

### 3.2 用户核心诉求
- 操作简单，不需要学习成本
- 上传后能快速出结果
- 抠图效果可接受
- 下载方便，不强制登录

---

## 4. 用户场景

### 场景 1：普通用户处理头像/照片
用户上传一张人物照片，希望快速获得透明背景 PNG，用于头像、海报或简历。

### 场景 2：电商用户处理商品图
用户上传商品图片，希望快速去背景，后续用于白底图或商品主图制作。

### 场景 3：内容创作者处理素材
用户上传图片，希望提取主体，用于封面、排版、视频素材。

---

## 5. MVP 功能范围

## 5.1 必做功能

### 5.1.1 图片上传
用户可以在首页上传单张图片。

支持方式：
- 点击上传
- 拖拽上传

支持格式：
- jpg
- jpeg
- png
- webp

限制：
- 单次仅允许上传 1 张图片
- 文件大小限制：建议 10MB 以内

---

### 5.1.2 自动去背景
前端上传图片后，服务端调用 Remove.bg API 进行处理，并返回去背景后的 PNG 图片。

要求：
- 用户无需额外设置参数
- 默认自动处理
- 处理过程需展示 loading 状态

---

### 5.1.3 结果预览
处理完成后，页面展示：
- 原图
- 去背景后的结果图

建议展示方式：
- 左右并排对比
或
- Before / After 切换

结果图需要支持透明背景预览（棋盘格背景）。

---

### 5.1.4 下载结果
用户可以下载处理后的透明背景 PNG 图片。

要求：
- 点击后直接下载
- 文件名可默认生成，例如：`removed-background.png`

---

### 5.1.5 错误提示
当出现异常时，页面应给出明确提示。

异常场景包括但不限于：
- 未上传文件
- 文件格式不支持
- 文件过大
- Remove.bg API 调用失败
- 网络超时
- 系统内部异常

提示原则：
- 文案清晰
- 不暴露敏感技术细节
- 可引导用户重试

---

## 5.2 建议做但不强制的 MVP 功能

### 5.2.1 基础限流
防止接口被滥用。

### 5.2.2 基础反爬 / 防刷
建议接入 Cloudflare Turnstile。

### 5.2.3 基础 SEO 信息
首页需具备：
- title
- meta description
- H1
- FAQ 内容块

---

## 5.3 明确不做功能

以下功能不纳入本期：

- 登录注册
- 保存上传记录
- 云端存储原图和结果图
- 批量上传/批量下载
- 换背景颜色
- AI 背景生成
- 图片编辑器
- 高清/普通清晰度分级
- 商业化支付流程
- 管理后台
- 邮件通知
- 多模型切换

## 7. 交互流程

## 7.1 主流程

1. 用户进入首页
2. 用户上传图片
3. 前端校验文件格式和大小
4. 前端调用后端接口
5. 页面展示处理中状态
6. 服务端调用 Remove.bg API
7. 返回结果图
8. 页面展示结果图
9. 用户点击下载

---

## 7.2 异常流程

### 异常 1：文件为空
提示：Please upload an image.

### 异常 2：格式不支持
提示：Unsupported file type. Please upload JPG, PNG, or WEBP.

### 异常 3：文件过大
提示：File is too large. Please upload an image smaller than 10MB.

### 异常 4：接口失败
提示：Failed to remove background. Please try again later.

### 异常 5：超时
提示：Processing is taking too long. Please retry.

---

## 8. 产品规则

### 8.1 上传规则
- 仅支持单张图片
- 文件类型限制为 jpg/jpeg/png/webp
- 文件大小限制为 10MB 以内

### 8.2 结果规则
- 输出格式统一为 PNG
- 结果仅在当前会话中存在
- 用户刷新或关闭页面后不保留

### 8.3 存储规则
- 本期不做数据库存储
- 不做对象存储
- 图片只在请求处理过程中以内存方式中转
- 不保留原图和结果图

---

## 9. 非功能需求

## 9.1 性能要求
- 页面首屏加载尽量轻量
- 单张图片处理尽量控制在数秒内
- 上传后应立即反馈处理中状态

## 9.2 可用性要求
- 支持桌面端浏览器
- 移动端至少可上传和查看结果
- 页面交互简单直观

## 9.3 安全要求
- Remove.bg API Key 不可暴露到前端
- API 必须通过服务端中转
- 增加基础防刷能力
- 不记录用户上传内容

## 9.4 稳定性要求
- 接口失败要有兜底提示
- 前端避免因为单次失败导致页面卡死
- 上传和处理过程中按钮状态明确

---

## 10. 技术实现要求

## 10.1 技术栈
- 前端：Next.js
- 样式：Tailwind CSS
- 部署：Cloudflare
- 抠图服务：Remove.bg API

## 10.2 后端接口
建议接口：

`POST /api/remove-background`

请求类型：
- `multipart/form-data`

字段：
- `image_file`

返回：
- 成功：图片流（`image/png`）
- 失败：JSON 错误信息

---

## 10.3 服务端逻辑
服务端需要完成：
- 文件存在校验
- 文件大小校验
- 文件类型校验
- 调用 Remove.bg API
- 接收结果并返回给前端
- 异常捕获和错误返回

## 13. 风险与限制

### 13.1 外部 API 依赖风险
项目完全依赖 Remove.bg API：
- 如果 API 异常，功能不可用
- 成本受调用量影响
- 速度和结果质量受第三方影响

### 13.2 无存储限制
不保存图片意味着：
- 无法查看历史结果
- 无法做后续二次编辑
- 无法支持用户回看

### 13.3 Cloudflare 环境限制
由于使用 serverless / edge 方式处理上传文件：
- 文件大小和处理时间需控制
- 不适合本期支持大文件或批量任务

---

## 14. 验收标准

满足以下条件即可视为 MVP 完成：

1. 用户可在首页上传单张图片
2. 系统能成功调用 Remove.bg API 去背景
3. 页面能展示处理后的结果图
4. 用户可下载 PNG 图片
5. 对格式错误、大小超限、接口失败有明确提示
6. Remove.bg API Key 不暴露到前端
7. 网站可在 Cloudflare 上正常部署访问
